1. Purpose of the document
The purpose of this style guide is to help localize content for Amazon in order to get parallel translations that will be used
for Machine Learning purposes.
To know more about parallel translation, read the Project Description carefully. If you have any doubts about the content
you are translating/post-editing, please submit an inquiry to your Project Manager by sharing the ASIN ID and source
content prior to delivering the final translation/post-editing.
Important note: If you find any discrepancy between this general style guide and your language-specific appendix, please
follow the instructions contained in this style guide.

2. Project Description
With “parallel translations”, we aim to provide a translation/post-editing which is as close to the source as possible. In
general, the guideline is to mirror the source, without changing its content or meaning.
We understand that some adjustments will have to be made, in order for the translation/post-editing to comply with
general grammar rules and with the syntactic structure of the target language, but whenever possible, try to keep the
same order as the source text (see examples below). Please note that no content should be added nor omitted in the
target.
The translations/post-edits are not meant to become part of website for the customers to read, but will be used for
Machine Learning purposes. This is the reason why we ask you to preserve as much source information as possible, while
ensuring that the target side is not less grammatical than the source side.
To summarize, in parallel translations:
1. The structure of the source must be kept as it is, as long as it doesn’t conflict with the target’s grammar and
syntactic rules.
2. Everything must be translated/post-edited. Do not add or omit any part of the source.
3. The punctuation of the source must be kept as it is (except when it conflicts with target’s grammar rules, i.e.: “12.5
cm” in English should be translated as “12,5 cm” in Spanish).
The jobs we will be sending you will be made up of individual products or ASINs (Amazon Standard Item Number e.g.
B00U2MTJA8); these are the actual products that can be found on the Amazon websites. ASIN content is made of three
main content types: product titles, product descriptions and bullet points (list of specifications about the product). If you
have doubts regarding the ASIN part you are translating/post-editing, please go to the Amazon website and check which
segments belong to the title, which ones to the bullet points and which ones to the description (please see the Checklist
below for instructions on how to find the product you are translating/post-editing on the website).

3. Checklist for General Translation/Post-Editing Process
Firstly, it is extremely important that all ASIN codes (beginning B0…) are transposed correctly, so please copy them
directly from the source to the target.
Translate/post-edit while making sure to adhere to the instructions and exceptions listed below.
While translating/post-editing, it can be extremely useful to see the context, the images and the layout of the text. In
order to access this kind of information you just have to enter the appropriate ASIN number on the source market’s
Amazon website as a search. If you cannot find a product by inserting the ASIN number in the search bar of the

4.2. Mistranslation
The target must accurately represent source content.
The terms provided by the memory matches or the term base (if any) may not always be the correct ones for the
given context; therefore, please make sure to double-check the output and make amendments where necessary.
For ambiguous terms, search for the ASIN of that product on the Amazon webpage search bar (see Checklist section)
of that particular product when you think you would need more clarification.

7.1. Measurement format
Convert units of measurement according to the standard unit system in your target language (e.g. inches into cm or vice-
versa). If two measurements are given (e.g. “12 inches / 30.5 cm”) keep both, for example in Spanish: “12 pulgadas / 30,5
cm”.
For monitors and other items where size is usually expressed in imperial (e.g. TVs, hard drives, laptops, bicycle tires, etc.),
keep units in the imperial system. If two measurements are given in the source (e.g. “ 12 inches / 30.5 cm”) keep both in
the target.
Converting clothing, shoe sizes, etc.: check the manufacturer’s website or use online sizing charts.
Other measurements: convert oz (into ml for liquids, g for non-liquids), gauge and ring sizes.

8.1. Company Style
Voice and Tone
Ensure to check the relevant language appendix (if available) in order to use the correct grammatical register, including
informal or formal pronouns and verbs, according to the requirements. If no appendix is provided, use the most
appropriate grammatical register for the given context and your target language.
Titles
Please follow the source without moving the content around. As an example, if the source title gives the color as the very
last info, keep it as the last info in the target title too.
Example [FR-EN]: Aztex Serviette Turban de Luxe, Coton, 64 x 23 cm, Gris → Aztex Luxury Hair Turban Towel, Cotton, 64 x
23cm, Slate
Title structure: strictly follow the order of elements in the source
Brand name missing in the source: it should be missing in the target
Product type missing in the source: it should be missing in the target
Brand names
Where there is an official translation/transliteration used on the manufacturers’ websites for the specific language, please
use this in the translation.
Otherwise, please use the source version. If the Brand Name is missing in the source, it should be missing in the target too.
Word order
Keep the same order as the source text wherever possible. Some adjustments will have to be made, in order for the
translation/post-editing to comply with general grammar rules and syntactic structure of the target language and
that’s OK.

9.3. Inconsistent with Termbase
You must always use any applicable terms from reference materials such as term bases (TB), if any, and adapt their syntax
to the context if needed. This will guarantee a consistent translation of key terms. The terms from the TB are marked in
yellow in the source segment, and are shown in the right side of the ATMS interface window (marked ‘TB’).
Term base prioritization
Since there might be several TBs attached to the same project, some terms may contradict one another, so please note
prioritization in ATMS. If more than one TB matches a term, the priority of the TB is displayed with each TB match in the
Search pane (see example in the screenshot below):
TB* – Primary TB
TB** – Secondary TB
TB – None (no priority)
Below are examples of TBs that might be attached to our projects:
Amazon Global TB: It is a multilingual glossary of both retail-specific terms and terms that are common across all
Amazon, such as Amazon brands and key programs and services names. Its purpose is to provide consistent
terminology translations across Amazon teams.
Acronym Central: It is a repository in the form of a term base with definitions for Amazon-specific acronyms. Its
purpose is to provide the expanded acronym as an additional context, which means the target term entry is always
untranslated. Some acronyms have multiple definitions (highlighted in yellow in the screenshot below). Please try to
make an educated guess based on the context before escalating inquiries.
If a term is included both in the Global TB and in the Acronym Central repository, you should always default to the
Global TB entry. You can check where a term comes from in the right-hand pane metadata (next to TB name,
highlighted in green).
https://amazontranslationlearn.com/mod/book/tool/print/index.php?id=729 31/32

Metadata
You will find the metadata for term entries in the bottom right corner of ATMS. It may include details like (but not limited
to): Definition, Part of speech, Usage Example. Read it carefully; it provides context for each term and will help you to
handle them correctly.
When in doubt, look for the TB name to identify the team or client from which it comes and use your judgement to
determine whether it applies to the segment you are working on before escalating enquiries. Note that the TB has priority
over the TM, so if you are provided with two possible translations, prioritize the TB.
